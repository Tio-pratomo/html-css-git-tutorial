## Materi: Pengetahuan & Konsep

Di sesi 17 kita tidak tambah komponen baru, tapi fokus **merapikan cara kamu memakai Bulma**: struktur HTML, semantik, aksesibilitas dasar, dan performa CSS.

Ini yang bikin proyek terasa “production‑ready”, bukan sekadar latihan.

Tiga fokus utama:

1. **HTML yang semantik & bersih** – membantu SEO, maintainability, dan aksesibilitas.
2. **Aksesibilitas dasar** untuk navbar, modal, dan notifikasi (ARIA, keyboard).
3. **Modular & performa** – kapan cukup CDN, kapan mulai modular import/mengurangi CSS.

### 1. HTML semantik + Bulma class (bukan sebaliknya)

Bulma memaksa kita tetap pakai HTML semantik (karena murni CSS):

- Gunakan elemen yang benar dulu, baru tambahkan class Bulma:
  - `<header>`, `<nav>`, `<main>`, `<aside>`, `<section>`, `<footer>`, `<article>`, `<ul>/<li>`, `<button>`, dsb.
- Hindari `<div>` di mana seharusnya `<button>` (misal untuk aksi), `<a>` untuk link, atau `<ul>` untuk list.

Contoh perbaikan (dari `div` generic jadi semantik):

```html
<!-- Kurang baik -->
<div class="navbar">...</div>

<!-- Lebih baik -->
<nav class="navbar" role="navigation" aria-label="main navigation">...</nav>
```

```html
<!-- Tombol filter -->
<button class="button is-small is-primary">Filter</button>
```

Best practice yang sering ditegaskan di panduan Bulma dan artikel best‑practice:

- Gunakan komponen Bulma **di atas HTML yang sudah benar**, jangan memaksa layout dengan class saja.
- List data yang bukan tabular → pakai `<ul>/<li>` + `media`/`box`, bukan `<table>`.

### 2. Aksesibilitas dasar: navbar, modal, notifikasi

#### Navbar

Navbar Bulma sudah menyertakan pattern ARIA dasar dalam dokumentasi:

- `role="navigation"` di `<nav>`.
- `aria-label="main navigation"` untuk memberi nama daerah navigasi.
- Burger button:
  - `role="button"`, `aria-label="menu"`, `aria-expanded="false"`, `data-target="id-menu"`.

**Pastikan:**

- Saat burger diklik dan `navbar-menu` menjadi aktif, kamu juga toggle `aria-expanded` dari `false` → `true` di JS (ini kecil tapi penting untuk screen reader).

#### Modal

Untuk modal lebih aksesibel (di luar hanya toggle `is-active`):

- Tambahkan `role="dialog"` dan `aria-modal="true"` pada container modal.
- Tambahkan `aria-labelledby` yang menunjuk ke elemen judul modal.

Contoh:

```html
<div
  id="modal-buku"
  class="modal"
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-buku-title"
>
  <div class="modal-background"></div>
  <div class="modal-card">
    <header class="modal-card-head">
      <p id="modal-buku-title" class="modal-card-title">Detail Buku</p>
      <button class="delete" aria-label="close"></button>
    </header>
    ...
  </div>
</div>
```

Level lanjut (kalau mau benar‑benar aksesibel):

- Saat modal dibuka: fokus dikirim ke elemen logis pertama di dalam modal (misal tombol “Beli Sekarang”).
- Trap focus di dalam modal sampai modal ditutup (ini butuh JS ekstra, bisa jadi materi sesi khusus nanti).

#### Notifikasi

Untuk notifikasi penting (error, sukses):

- Gunakan `role="alert"` atau `role="status"` sesuai konteks, agar dibaca screen reader tanpa perlu fokus manual.
- Contoh:

```html
<div
  id="welcome-notification"
  class="notification is-info is-light"
  role="status"
>
  ...
</div>
```

Untuk error penting (misal “gagal menyimpan”): gunakan `role="alert"`.

### 3. Performa & modularitas CSS

Beberapa poin dari artikel evaluasi Bulma & best practice penggunaan:

- **CDN full Bulma** (sekitar 200KB minified) cukup untuk:
  - Prototipe.
  - Project kecil / internal.
  - Belajar (kasus kamu sekarang).
- Saat aplikasi membesar atau kamu peduli ukuran CSS:
  - Mulai pikirkan **modular Sass import** untuk hanya menyertakan komponen yang dipakai.
  - Atau gunakan **tooling** (PurgeCSS, uncss) untuk membuang class yang tidak terpakai (advanced, di luar scope awal).

Prinsip praktis:

- Jangan premature optimization: untuk belajar dan project kecil, CDN + sedikit custom CSS/variables sudah sangat oke.
- Begitu kamu merasa CSS mulai gemuk dan pattern layout sudah stabil, baru worth pindah ke pipeline Sass modular (Sesi 15).

## Praktik (Refactor Ringan di Project Kamu)

1. **Semantik**
   - Cek tiap halaman utama dan ganti `<div>` dengan elemen semantik yang lebih tepat (nav, main, section, footer, ul/li, button).
   - Pastikan tabel hanya dipakai ketika data benar‑benar tabular; list buku biasa boleh pakai `media` + `ul/li` + `box`.

2. **ARIA & roles**
   - Tambahkan `role` dan `aria-*` minimal untuk:
     - Navbar: `role="navigation"`, `aria-label`.
     - Modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`.
     - Notifikasi penting: `role="alert"` atau `role="status"`.

3. **Script kecil untuk `aria-expanded`**
   - Update handler burger di `main.js` untuk toggle `aria-expanded` bersamaan dengan `is-active`.
