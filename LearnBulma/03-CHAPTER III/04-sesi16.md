## Materi: Pengetahuan & Konsep

Di sesi ini kita “merangkai” semua yang sudah kamu pelajari jadi satu **mini project**: _Online Book Publisher Dashboard_ berbasis Bulma.

**Fokusnya:**

- Kamu paham **bagian mana pakai apa** (hero, navbar, menu, columns, grid, table, form, notification, card, modal, tabs, JS, tema), bukan cuma potongan terpisah.
- Struktur proyek rapi, jadi pondasi kalau nanti mau dihubungkan ke backend/API.

### 1. Struktur file proyek

Usulkan struktur direktori simpel:

```text
bulma-book-dashboard/
├─ index.html          -> redirect / landing kecil ke login.html
├─ login.html          -> Sesi 2 (hero + form login)
├─ dashboard.html      -> Sesi 3 + 4 + 8 (navbar, sidebar, notifikasi, cards)
├─ inventory.html      -> Sesi 6 + 7 (tabel + quick edit)
├─ add-book.html       -> Sesi 5 (breadcrumb + form + file upload)
├─ interactive.html    -> Sesi 9 (modal + tabs demo, bisa di-embed ke dashboard)
├─ theme-demo.html     -> Sesi 14 (theme & dark mode demo, optional)
├─ columns-demo.html   -> Sesi 11–12 (playground columns, optional)
├─ grid-demo.html      -> Sesi 13 (playground grid, optional)
├─ css/
│  └─ custom.css       -> helper kecil JS/tema kalau perlu (opsional, bisa inline)
└─ js/
   └─ main.js          -> kumpulan script JS (navbar burger, modal, tabs, theme)
```

File `*-demo.html` sifatnya playground; yang dianggap “produk” adalah:

- `login.html`
- `dashboard.html`
- `inventory.html`
- `add-book.html`

Nanti kalau mau, kamu bisa merge sebagian interaktivitas dari `interactive.html` dan tema dari `theme-demo.html` ke `dashboard.html` supaya dashboard jadi benar‑benar hidup.

### 2. Peran tiap halaman (yang “utama”)

Ringkas fungsi utama per halaman:

- `login.html`
  - Hero fullscreen, form login, helpers typography & spacing.
- `dashboard.html`
  - Navbar + burger + JS toggle.
  - Sidebar menu vertikal.
  - Notifikasi welcome (dismissable) + grid cards “Latest Additions”.
  - Bisa juga ditambah tile stats (Sesi 8) dan/atau tab kecil di dalam konten.
- `inventory.html`
  - Toolbar level + dropdown filter.
  - Tabel inventory (striped + hoverable + fullwidth + `table-container`).
  - Inline editing quick stock (Sesi 7) di halaman terpisah `quick-edit.html` _atau_ digabung ke inventory sebagai section kedua.
- `add-book.html`
  - Breadcrumb.
  - Form kompleks + file upload cover + JS untuk nama file.

Dengan layout tersebut, kamu sudah menutupi hampir semua komponen besar di dokumentasi Bulma: layout, elements, components, form, grid, helpers, dan sedikit integrasi JS.

## Praktik: Menyatukan Script JS Umum

Supaya rapi, kita satukan behavior JS dasar ke satu file `js/main.js` dan include di semua halaman “utama”.

### 1. `js/main.js` – navbar burger, modal, tabs, file input, theme toggle

Isi minimal yang mencakup sesi 3, 5, 9, 14 (boleh kamu trim kalau belum dipakai):

```js title="js/main.js"
document.addEventListener("DOMContentLoaded", () => {
  // NAVBAR BURGER
  const navbarBurgers = Array.from(document.querySelectorAll(".navbar-burger"));
  navbarBurgers.forEach((el) => {
    el.addEventListener("click", () => {
      const target = el.dataset.target;
      const $target = document.getElementById(target);
      el.classList.toggle("is-active");
      $target.classList.toggle("is-active");
    });
  });

  // FILE INPUT NAME (add-book)
  const fileInput = document.querySelector(".file-input");
  const fileName = document.querySelector(".file-name");
  if (fileInput && fileName) {
    fileInput.addEventListener("change", () => {
      if (fileInput.files.length > 0) {
        fileName.textContent = fileInput.files[0].name;
      } else {
        fileName.textContent = "No file selected";
      }
    });
  }

  // MODAL (interactive / bisa dipakai di dashboard)
  function openModal($el) {
    $el.classList.add("is-active");
  }
  function closeModal($el) {
    $el.classList.remove("is-active");
  }
  function closeAllModals() {
    document.querySelectorAll(".modal").forEach(($modal) => {
      closeModal($modal);
    });
  }

  document.querySelectorAll(".js-modal-trigger").forEach(($trigger) => {
    const modalId = $trigger.dataset.target;
    const $target = document.getElementById(modalId);
    if ($target) {
      $trigger.addEventListener("click", () => openModal($target));
    }
  });

  document
    .querySelectorAll(
      ".modal-background, .modal-close, .modal-card-head .delete, .modal-card-foot .button",
    )
    .forEach(($close) => {
      const $target = $close.closest(".modal");
      if ($target) {
        $close.addEventListener("click", () => closeModal($target));
      }
    });

  document.addEventListener("keydown", (event) => {
    if (event.key === "Escape") {
      closeAllModals();
    }
  });

  // TABS (interactive)
  const tabs = document.querySelectorAll(".tabs li");
  const tabContentBoxes = document.querySelectorAll("#tab-content > div");
  if (tabs.length && tabContentBoxes.length) {
    tabs.forEach((tab) => {
      tab.addEventListener("click", () => {
        tabs.forEach((item) => item.classList.remove("is-active"));
        tabContentBoxes.forEach((box) => {
          box.classList.add("is-hidden");
          box.classList.remove("is-active");
        });
        tab.classList.add("is-active");
        const target = tab.dataset.tab;
        const targetContent = document.querySelector(
          `[data-content="${target}"]`,
        );
        if (targetContent) {
          targetContent.classList.remove("is-hidden");
          targetContent.classList.add("is-active");
        }
      });
    });
  }

  // THEME TOGGLE (opsional, kalau ada tombol #theme-toggle)
  const themeToggle = document.getElementById("theme-toggle");
  if (themeToggle) {
    themeToggle.addEventListener("click", () => {
      document.documentElement.classList.toggle("theme-dark");
    });
  }
});
```

Lalu di tiap HTML utama (`dashboard.html`, `inventory.html`, `add-book.html`, `interactive.html`), sebelum `</body>`:

```html
<script src="js/main.js"></script>
```

Dengan ini:

- Satu file JS meng-handle behavior generik Bulma (burger, modal, tabs, file input).
- Kamu tidak copy–paste script di tiap halaman lagi.

## Checklist Proyek Mini (yang perlu kamu capai)

Kalau kamu ingin menganggap “seri Bulma level 1” ini selesai dengan mantap, gunakan checklist ini:

- [ ] `login.html` jalan, responsif, form di tengah, layout hero jelas.
- [ ] `dashboard.html` punya:
  - Navbar + burger yang berfungsi.
  - Sidebar menu vertikal.
  - Satu notifikasi di atas yang bisa ditutup.
  - Grid card “Latest Additions” atau sejenis.
- [ ] `inventory.html` punya:
  - Toolbar (level) + dropdown filter status.
  - Tabel inventory pakai `table-container`, `is-striped`, `is-hoverable`.
- [ ] `add-book.html` punya:
  - Breadcrumb.
  - Form lengkap + file upload dengan nama file tampil.
- [ ] `interactive.html` (atau di-embed di dashboard) punya:
  - Minimal 1 modal yang bisa dibuka/tutup dengan tombol + ESC.
  - Tabs yang bekerja.
- [ ] Tema dasar diset lewat CSS variables (di salah satu file head) dan kamu paham cara mengubah warna primary tanpa ubah HTML.
