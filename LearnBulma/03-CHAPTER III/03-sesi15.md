## Materi: Pengetahuan & Konsep

Untuk proyek kecil, kustomisasi via **CSS variables** (Sesi 14) sudah cukup. Tapi kalau kamu mulai pakai build tools (npm, bundler), cara yang lebih **powerful** adalah lewat **Sass**: override variable Bulma sebelum CSS-nya dikompilasi.

**Dengan Sass kamu bisa:**

- Ganti warna, font, breakpoint, dsb. lewat **Sass variables** (`$primary`, `$family-sans-serif`, `$tablet`, dll.).
- Import **hanya komponen yang dibutuhkan** (modular Sass) supaya CSS final lebih kecil.
- Pakai **responsive mixins** Bulma (`from()`, `until()`, `mobile`, `tablet`, dll.) supaya media query kamu sinkron dengan breakpoint Bulma.

Anggap sesi ini sebagai “track lanjutan”, bukan keharusan untuk lanjutin materi lain.

### 1. Alur kerja dasar: Bulma + Sass

Ringkasannya (sesuai dokumentasi “Customize with Sass”):

1. Install `sass` dan `bulma` via npm.
2. Buat file Sass kamu sendiri (misal `my-bulma-project.scss`).
3. Di situ, kamu:
   - Set **Sass variables** kamu (misalnya warna primary, font).
   - Import Bulma dengan `@use` + `with` untuk override variable.
4. Compile `.scss` → `.css` dengan script npm (`sass my-bulma-project.scss my-bulma-project.css`).

Struktur paling sederhana:

```scss title="my-bulma-project.scss"
// 1. Override variable SEBELUM import Bulma
$primary: #1e88e5;
$family-sans-serif:
  "Inter",
  system-ui,
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  sans-serif;

// 2. Import Bulma dengan @use (v1)
@use "bulma/bulma";
```

Setelah dikompilasi, semua komponen Bulma akan pakai warna & font baru kamu karena mereka bergantung ke Sass variables global.

### 2. Modular Sass: import hanya yang kamu perlu

Dokumentasi “Customize with Modular Sass” menjelaskan cara import **hanya sebagian** Bulma, misalnya **utilities** + **form** + **navbar** saja.

Contoh pola (disederhanakan):

```scss
// variables: atur nilai dulu
$primary: #00b894;
$info: #0984e3;
$family-sans-serif: "Inter", sans-serif;

// utilities: breakpoints, mixins, helpers
@use "bulma/sass/utilities" with (
  $primary: $primary,
  $info: $info,
  $family-sans-serif: $family-sans-serif
);

// layout & components yang kamu butuh
@use "bulma/sass/base";
@use "bulma/sass/elements/button";
@use "bulma/sass/elements/title";
@use "bulma/sass/components/navbar";
@use "bulma/sass/components/card";
@use "bulma/sass/form";
```

Keuntungan:

- CSS final lebih kecil daripada `bulma.min.css` full karena kamu hanya ambil bagian yang dipakai.
- Mudah membuat “theme pack” sendiri untuk beberapa proyek sekaligus.

Ini cocok kalau nanti kamu sudah nyaman dengan bundler (Vite, Webpack, esbuild) dan ingin kontrol penuh atas CSS.

### 3. Responsive mixins Bulma (Sass)

Tanpa mixin, kamu tulis media query sendiri, misalnya:

```scss
@media screen and (min-width: 1024px) {
  .my-box {
    padding: 3rem;
  }
}
```

Masalah: kalau breakpoints Bulma berubah, kamu harus update semua media query manual. Bulma menyediakan **responsive mixins** di Sass supaya breakpoint **sinkron** dengan yang dipakai framework.

Mixins dasar yang penting:

- `from($breakpoint)` → style berlaku mulai lebar tertentu ke atas.
- `until($breakpoint)` → style berlaku sampai lebar tertentu ke bawah.
- Named mixins: `mobile`, `tablet`, `desktop`, `widescreen`, `fullhd`, dll.

Contoh:

```scss
.my-box {
  background: red;

  @include from(desktop) {
    background: blue;
  }
}
```

**Artinya:**

- Di mobile dan tablet: background merah.
- Di desktop ke atas: background biru (karena `from(desktop)`).

Contoh dengan named mixin:

```scss
.my-title {
  font-size: 1.25rem;

  @include tablet {
    font-size: 1.5rem;
  }

  @include desktop {
    font-size: 2rem;
  }
}
```

- Di mobile: 1.25rem.
- Tablet: 1.5rem.
- Desktop ke atas: 2rem.

Ini sangat berguna kalau kamu punya komponen kostum di luar komponen Bulma (misalnya `.my-card` sendiri), tapi ingin responsifnya **konsisten** dengan seluruh sistem Bulma.

## Praktik (Opsional, untuk yang siap pakai npm)

Kalau kamu mau coba step by step:

### Latihan 1: Override warna & font via Sass

1. Buat folder proyek baru, `npm init -y`.
2. `npm install bulma sass`.
3. Buat file `my-bulma-project.scss` seperti contoh di atas (override `$primary`, `$family-sans-serif`).
4. Tambah script di `package.json`:

   ```json
   "scripts": {
     "build:css": "sass my-bulma-project.scss my-bulma-project.css --no-source-map",
     "watch:css": "sass --watch my-bulma-project.scss:my-bulma-project.css --no-source-map"
   }
   ```

5. Jalankan `npm run build:css`, lalu di HTML kamu pakai:

   ```html
   <link rel="stylesheet" href="my-bulma-project.css" />
   ```

Cek: seluruh tema Bulma sudah pakai warna & font baru.

### Latihan 2: Pakai mixin `tablet` untuk komponen kustom

Di `my-bulma-project.scss`, setelah `@use "bulma/bulma";`, tambahkan:

```scss
.my-custom-card {
  padding: 1rem;
  background: white;

  @include tablet {
    padding: 2rem;
  }

  @include desktop {
    padding: 3rem;
  }
}
```

**Lalu di HTML:**

```html
<div class="box my-custom-card">Uji padding responsif dengan resize layar.</div>
```

Resize dan lihat perubahan padding di tablet vs desktop.
