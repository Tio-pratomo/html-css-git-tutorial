## Materi: Pengetahuan & Konsep

Di sesi 11 kamu sudah pegang dasar `columns` + `column` dan sistem ukuran 12-grid.

Sesi 12 ini fokus ke:

- Bagaimana columns berubah di tiap breakpoint (mobile, tablet, desktop, dst.).
- Mengatur jarak antar kolom (gap, gapless, variable gap).
- Nesting columns (columns di dalam column) untuk layout yang lebih fleksibel.

### 1. Responsiveness dasar columns

Bulma itu **mobile-first**. Untuk columns:

- Secara default, di **mobile (< 768px)**, columns akan **stack vertikal** (satu di atas yang lain).
- Layout horizontal (sejajar) baru aktif mulai breakpoint `tablet` ke atas.

**Kalimat penting dari dokumentasi :**

> “By default, columns are only activated from tablet onwards. This means columns are stacked on top of each other on mobile.”

Kalau kamu mau **paksa columns tetap horizontal di mobile**, tambahkan `is-mobile` di container:

```html
<div class="columns is-mobile">
  <div class="column">A</div>
  <div class="column">B</div>
</div>
```

- Tanpa `is-mobile`: di HP, A dan B akan jadi dua baris; di tablet/desktop baru sejajar.
- Dengan `is-mobile`: di HP pun A dan B tetap sejajar (hati-hati, bisa jadi sempit).

### 2. Ukuran berbeda per breakpoint

Kamu bisa memberi **ukuran kolom berbeda-beda per breakpoint**, misalnya:

- Mobile: full width (12/12).
- Tablet: 8/12 (2/3).
- Desktop: 6/12 (1/2).
- Widescreen: 4/12 (1/3).

Pola umum:

- `is-{size}-{breakpoint}`
  - Contoh: `is-12-mobile`, `is-8-tablet`, `is-6-desktop`, `is-4-widescreen`.

Contoh nyata:

```html
<div class="columns">
  <div
    class="column
           is-12-mobile
           is-8-tablet
           is-6-desktop
           is-4-widescreen
           has-background-primary-light"
  >
    Responsif per breakpoint
  </div>
</div>
```

Artinya:

- Di mobile: lebar 12/12 (full).
- Di tablet: lebar 8/12 (2/3).
- Di desktop: lebar 6/12 (1/2).
- Di widescreen: lebar 4/12 (1/3).

Kamu juga bisa pakai bentuk fractional: `is-half-tablet`, `is-one-third-desktop`, `is-one-quarter-fullhd`. Prinsipnya sama, hanya beda nama helper.

### 3. Columns options: alignment, multiline, dll.

Beberapa modifier penting di container `columns`:

- `is-vcentered`: semua kolom akan align vertical-center dalam satu baris (berguna kalau tinggi konten beda-beda).
- `is-centered`: kolom-kolom akan di-center secara horizontal (bukan mepet kiri).
- `is-multiline`: mengizinkan kolom membungkus ke baris berikutnya (wrap) saat tidak muat satu baris.
- `is-mobile`, `is-desktop`: memaksa behavior tertentu hanya di breakpoint tertentu.

Contoh:

```html
<div class="columns is-vcentered is-multiline">
  <div class="column is-3">Sidebar</div>
  <div class="column">
    <p>Konten yang mungkin lebih tinggi dari sidebar.</p>
  </div>
</div>
```

### 4. Columns gap: default, gapless, variable gap

Setiap column punya **gap** di kiri dan kanan sebesar variabel `$column-gap` (default 0.75rem), sehingga jarak antar dua kolom (gap di tengah) = 1.5rem.

**Tiga konsep penting:**

1. **Default gap**
   - Tidak perlu class khusus; ini yang kamu lihat selama ini.

2. **Gapless** – tanpa jarak sama sekali
   - Tambahkan `is-gapless` di `columns` untuk menghilangkan semua gap.

   ```html
   <div class="columns is-gapless">
     <div class="column has-background-primary">1</div>
     <div class="column has-background-link">2</div>
     <div class="column has-background-info">3</div>
   </div>
   ```

   - Bisa digabung dengan `is-multiline` untuk grid rapat tanpa celah.

3. **Variable gap** – custom jarak antar kolom
   - Di Bulma modern, kamu bisa pakai `is-{n}` di container `columns` untuk mengatur gap (0–8).
   - Contoh umum:
     - `is-0` → tanpa gap (seperti `is-gapless`).
     - `is-3` → default (0.75rem per sisi, 1.5rem antar kolom).
     - `is-8` → gap maksimum (~2rem per sisi).

**Contoh:**

```html
<div class="columns is-variable is-5">
  <div class="column has-background-primary-light">A</div>
  <div class="column has-background-link-light">B</div>
  <div class="column has-background-info-light">C</div>
</div>
```

- `is-variable is-5` → aktifkan “variable gap” dan pakai skala `5` (sekitar 1.25rem per sisi, 2.5rem antar kolom, bergantung konfigurasi Bulma).

Ada juga **breakpoint-based column gaps** — misalnya gap besar di desktop tapi kecil di mobile — via modifier tambahan per breakpoint, tapi untuk sekarang cukup paham konsep dasar variable gap dulu.

### 5. Nesting columns (columns di dalam column)

Bulma mengizinkan columns di-nest di dalam column supaya layout bisa sangat fleksibel.

Struktur resmi:

```html
<div class="columns">
  <div class="column">
    <!-- Nested columns -->
    <div class="columns">
      <div class="column">Nested 1</div>
      <div class="column">Nested 2</div>
    </div>
  </div>

  <div class="column">Kolom kanan biasa</div>
</div>
```

Aturan penting:

- Struktur: `columns` (outer) → `column` → `columns` (nested) → `column` …
- Jangan langsung `columns` di dalam `columns` tanpa pembungkus `column`.
- Nesting + responsiveness membuat layout sangat fleksibel (misal: sidebar + konten, di dalam konten masih ada grid 3 kolom).

## Praktik

Kita lanjutkan `columns-demo.html` dari Sesi 11 agar jadi playground responsif.

### Langkah 1: Responsive column per breakpoint

Di dalam `<div class="container">`, tambahkan:

```html
<h2 class="title is-4 mt-5">Responsif per Breakpoint</h2>

<div class="columns">
  <div
    class="column
           is-12-mobile
           is-8-tablet
           is-6-desktop
           is-4-widescreen
           has-background-primary-light"
  >
    is-12-mobile / is-8-tablet / is-6-desktop / is-4-widescreen
  </div>
  <div class="column has-background-link-light">Kolom sisa (auto)</div>
</div>
```

Buka DevTools, aktifkan mobile/tablet/desktop mode, dan perhatikan perubahan lebar kolom pertama di tiap breakpoint.

### Langkah 2: `is-mobile` vs default behavior

Tambahkan dua blok untuk dibandingkan:

```html
<h2 class="title is-4 mt-5">Default vs is-mobile</h2>

<p class="has-text-weight-semibold">Default (stack di mobile)</p>
<div class="columns">
  <div class="column has-background-info-light">A</div>
  <div class="column has-background-success-light">B</div>
</div>

<p class="has-text-weight-semibold mt-4">
  is-mobile (tetap horizontal di mobile)
</p>
<div class="columns is-mobile">
  <div class="column has-background-info-light">A</div>
  <div class="column has-background-success-light">B</div>
</div>
```

Resize ke lebar HP dan lihat bedanya: blok pertama stack vertikal, blok kedua tetap sejajar.

### Langkah 3: Gapless & variable gap

Tambahkan:

```html
<h2 class="title is-4 mt-5">Gapless & Variable Gap</h2>

<p class="has-text-weight-semibold">Gapless</p>
<div class="columns is-gapless">
  <div class="column has-background-danger-light">1</div>
  <div class="column has-background-primary-light">2</div>
  <div class="column has-background-link-light">3</div>
  <div class="column has-background-info-light">4</div>
</div>

<p class="has-text-weight-semibold mt-4">Variable gap (is-variable is-6)</p>
<div class="columns is-variable is-6">
  <div class="column has-background-success-light">A</div>
  <div class="column has-background-warning-light">B</div>
  <div class="column has-background-info-light">C</div>
</div>
```

**Bandingkan visual gap antara:**

- Default columns (dari Sesi 11).
- Gapless: benar-benar menempel.
- Variable gap `is-6`: jarak lebih besar dari default.

### Langkah 4: Nesting sederhana

Tambahkan:

```html
<h2 class="title is-4 mt-5">Nesting Columns</h2>

<div class="columns">
  <!-- Kolom kiri berisi nested columns -->
  <div class="column has-background-light">
    <p class="has-text-weight-semibold mb-2">Kiri (nested)</p>
    <div class="columns is-gapless">
      <div class="column has-background-primary-light">Nested 1</div>
      <div class="column has-background-link-light">Nested 2</div>
    </div>
  </div>

  <!-- Kolom kanan biasa -->
  <div class="column has-background-warning-light">
    <p class="has-text-weight-semibold">Kanan</p>
    <p>Tanpa nesting.</p>
  </div>
</div>
```

Perhatikan:

- Outer `columns` membagi halaman jadi 2.
- Di kolom kiri, inner `columns` membagi area kiri lagi menjadi 2 kolom kecil.

- `is-{size}-{breakpoint}` untuk lebar responsif.
- `is-mobile` untuk memaksa columns horizontal di HP.
- `is-gapless`, `is-variable is-N` untuk kontrol jarak.
- Nesting `columns` di dalam `column`.

kita bisa lanjut di **Sesi 13** ke **Grid system (Smart Grid / Fixed Grid)** yang lebih powerful untuk layout 2D atau melanjutkan ke **customization (CSS variables / Sass)**, tergantung kamu mau fokus ke mana dulu.
