## Materi: Pengetahuan & Konsep

Di sesi 19 kita rapikan dulu **gambaran besar**: apa yang sudah kamu kuasai, pola apa yang penting untuk diulang, dan bagaimana latihan mandiri setelah seri ini.

Secara garis besar, kamu sudah menyentuh **hampir semua area utama Bulma**:

- layout (hero, section, container, columns, grid),
- elements (title, button, form, table, notification),
- components (navbar, card, breadcrumb, modal, tabs, dropdown),
- helpers (spacing, typography, flex, visibility),
- interaktivitas dasar dengan Vanilla JS,
- plus kustomisasi tema pakai CSS variables & sedikit Sass.

### 1. Peta kemampuan yang sudah kamu punya

Diringkas dari dokumentasi resmi dan panduan praktis:

- **Layout & struktur**
  - `hero` (login, landing), `section` + `container` (dashboard & landing), `columns` + responsivenya, `grid` (smart & fixed) untuk layout 2D.
- **Elements & components**
  - Judul (`title`), teks (`content`), `box`, `button`, `tag`, `notification`, `table`, `image`.
  - Navbar, breadcrumb, card, dropdown, modal, tabs, pagination, menu/sidebar.
- **Forms & data**
  - Input, textarea, select, checkbox, radio, file upload dengan JS kecil untuk nama file.
  - Tabel biasa + tabel dengan inline form (quick edit).
- **Helpers & responsif**
  - Spacing (`mt-*`, `mb-*`, `p-*`), teks (`has-text-*`, `is-size-*`, weight), warna, visibility (`is-hidden-*`), flex helpers.
- **Interaktivitas (Vanilla JS)**
  - Navbar burger, dropdown klik, modal buka/tutup + ESC, tabs switch, notifikasi dismiss, dark mode toggle sederhana.
- **Kustomisasi tema**
  - Override warna & scheme via CSS variables (`--bulma-primary-*`, `--bulma-scheme-main`, dll.), dan gambaran umum cara pakai Sass untuk override `$primary`, import modular, dan responsive mixins.

Ini kira‑kira set kemampuan yang juga jadi target di kursus/bootcamp Bulma modern dan buku resmi Bulma (layouts, components, kombinasi antar komponen, integrasi JS).

### 2. Pola yang paling penting diulang

Beberapa pattern yang menurut banyak panduan Bulma layak kamu ulang berkali‑kali:

- **Pola halaman utuh**
  - Login: hero + box + form.
  - Dashboard: navbar + sidebar + section konten (tabel, card, notifikasi).
  - Landing: hero fullheight + beberapa section + footer.
- **Pola layout**
  - `section > container > columns > column` untuk blok biasa.
  - `hero-head / hero-body / hero-foot` untuk layar pertama landing.
  - `columns` (1D) vs `grid` (2D) – kapan pakai mana.
- **Pola data + aksi**
  - Tabel list + tombol aksi (edit/delete) di kanan, dengan `table-container` untuk mobile.
  - Inline editing: gabungkan `input is-small`, `field has-addons`, `tag` status, dan helper warna.
- **Pola interaktivitas**
  - Toggling class (`is-active`, `is-hidden`) untuk semua komponen interaktif.
  - “Script generik” yang bekerja untuk banyak instance (modal, tabs) daripada JS khusus satu elemen.

Semakin sering kamu mempraktikkan pola ini, semakin otomatis kamu membangun halaman baru hanya dengan menggabung‑gabungkan blok.

### 3. Strategi latihan mandiri setelah seri ini

Dari panduan belajar Bulma yang baru dan kursus gratis di luar sana, pola latihannya kurang lebih sama:

- **1. Cloning kecil**
  - Pilih 1–2 UI sederhana (misalnya “dashboard analytics sederhana” atau “landing SaaS minimalis”), lalu coba tiru layout dan gaya visualnya pakai Bulma.
  - Fokus: hero, section, card, tabel, form.

- **2. Variasi atas project yang sudah kamu punya**
  - Ubah tema: mainkan CSS variables (warna, background, text).
  - Refactor layout: sebagian yang pakai `columns` bisa kamu coba ganti dengan Smart Grid / Fixed Grid.
  - Tambah fitur kecil: misalnya notifikasi auto‑hide, atau card hover effect.

- **3. Tambah 1–2 resource rujukan**
  - Dokumentasi Bulma (yang sudah kamu pakai) + 1 kursus pendek (Scrimba / YouTube) itu sudah cukup sebagai referensi jika kamu lupa sintaks class tertentu.

:::note[Kuncinya]
jangan terlalu lama di materi; lebih baik sering bikin layout baru kecil dengan komponen yang sama.
:::
